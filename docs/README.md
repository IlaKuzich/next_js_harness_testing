# docs

reference documentation for the project. start with
[`ARCHITECTURE.md`](./ARCHITECTURE.md) for the big picture, then dive into a
domain spec under [`specs/`](../specs):

- [orders](../specs/orders.md) — order creation, totals, and the status lifecycle.
- [reviews](../specs/reviews.md) — product reviews, ratings, and their rules.
- [payments](../specs/payments.md) — polar customers, subscriptions, and checkout.

for setup and contribution workflow see [`CONTRIBUTING.md`](./CONTRIBUTING.md).
for the running list of changes see [`CHANGELOG.md`](./CHANGELOG.md).
